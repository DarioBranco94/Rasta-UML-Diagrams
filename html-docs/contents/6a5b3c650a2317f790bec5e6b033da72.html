Description

This use case summarizes the interaction between an administrator (Admin) and the Tourism Flow GUI. The user must authenticate to the system before interacting with it. Then, she can analyze data about tourism flows, by means of an interactive visual interface. Optionally, the user can ask the system to produce reports of various types on the analyzed data (e.g., reports with summarizing text and/or charts).
