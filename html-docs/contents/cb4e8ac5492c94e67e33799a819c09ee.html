Description

This sequence diagram shows a common sequence of interactions of the user with the system for filtering the list of POIs displayed on the frontend. 1. the user starts the sequence requesting the list of POIs that will be returned unfiltered for the first time. 2. The frontend will return the list of POIs with categories (added by the POI-TOI scorer) 3. the user can activate and deactivate the filters and the frontend will automatically refresh the list of POIs included in the selected categories.

This sequence is valid for a client-side filtering feature but can be replicated also with a server-side filtering functionality.
