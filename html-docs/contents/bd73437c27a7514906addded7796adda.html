Description

This use case involves four different actors and it groups different use cases. The Decision Maker is a type of user that is between the domain expert and the administrator. This user may have access to reserved areas and features of the system that are not allowed to the user but allowed to the administrators. The main objective of the decision maker is to use the system to make decisions about the overall management of Points of Interests. This user relates with the POI and TOI Management use case that groups the functions for managing the set of information related to POIs and TOIs stored in the system. This group of functions are reserved to the administrator and decision maker and requires authentication. The POI and TOI Management generalizes the Create POI and Edit/Delete POI use cases. The Create POI functionality allows the creation of a new POI in the system. The admin creates the POI, by filling the required information as geographical location, category, description etc. The Edit/Delete POI functionality, instead, allows the modification of a POI in the system. Both functionalities (Create, Edit/Delete POI) can be extended with a Semantic Enhancement. This functionality allows the POI textual description in the system to be enhanced with semantic links. While the admin creates or edits the POI, by filling all the required information, the system will analyze the corresponding fields to identify semantic links to recognized entities. To this aim, the Semantic Enhancement includes the Semantic Annotation use case, which is responsible for annotating a text with semantic links linking strings to external resources, e.g., wikipedia or dbpedia pages. The Admin actor is responsible for the Configure Semantic Enrichment use case. This function allows the semantic engine to be configured with basically the following information: multiple enhancement chains, selected enhancement engine, languages, and other minor configurations. Overall, the Admin is a user with privileges to interact with the reserved parts of the system, including configuration and data-entry that are not allowed to the main users of the system.

The Tourist actor is a special type of user mainly interested in functionalities of the system offering services for navigation, information and selection of POIs. This user relates with the Read POI Information use case, enabling the user to read information about a selected POI. This use case can be activated by selecting a POI of the list on the frontend, as described by the Select POI use case. Indeed, it includes the List POIs use case that permits users to navigate through the list of POIs. While reading the description and information about the POI, the user can follow the links in the text permitting to navigate to an external resource (e.g. wikipedia, external website). This last functionality is managed by the Navigate to External Resource use case use, which represents a function that following a link in a description or text navigates to an external resource. This use case is optional since the user can also decide to skip it. The Read POI Information use case is optional since the user can be satisfied about the provided information on the frontend. The list of POIs displayed on the frontend can be refined, as described by the Filter POIs use case. Indeed, if the list includes many results the user may want to refine the results by considering his/her preferences. A categorization of the POI can offer an effective way of filtering them.

Moreover, the users of type Admin can perform the Tourism Flows Analysis use case. It requires authentication, as described by the Authentication use case, and exploits interactive visual interfaces. It can be extended with facilities to automatically generate reports, as described by the Production of Reports use case. This use case represents functions for automatically generating reports to better help decision makers to understand data. Lastly, the Tourism Flow Analysis includes the Generation of Dashboard use case. It represents functions for generating interactive dashboards automatically to further help decision makers for the data analysis. This functionality is one of the options supporting the tourism flow analysis and it mainly generates charts and gauges that can be consulted only online and when logged into the system.
